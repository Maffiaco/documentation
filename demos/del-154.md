### [TP] [DELIANY-3243](https://deliany.youtrack.cloud/agiles/131-3/current?issue=DELIANY-3243) [Backoffice] Invoice display enhancement for Orders/Bookings
  - ## Issue
- The existing **bill module** had a rigid structure.
- Billing-related types were scattered across the codebase.
- The system had limited support for configuring different document displays.
- Adding or updating document types (e.g., invoices, food checks) required changes in multiple places.
- This increased complexity and created technical debt.

## Solution
- Refactored the bill module for better structure and maintainability.
- Introduced **InvoiceV2** and **FoodCheckV2** for improved document generation.
- Added **InvoiceDisplayConfig** to support flexible display configurations.
- Unified billing-related types into a shared `billType`.
- Switched document generation to a **configuration-driven approach** for easier extensibility.
