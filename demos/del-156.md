## Sprint DEL-156


### [TP] [DELIANY-3265](https://deliany.youtrack.cloud/agiles/131-3/current?issue=DELIANY-3265) [BackOffice] Add export action permission to configure export data

**Issue**  
Currently, the system does not have a clear permission control for the Export Excel action in CRM Persons. This may allow unauthorized users to export sensitive customer data or cause inconsistency with access control.

**Solution**  
- Add configurable permission for Export Excel action  
- Only users with permission can export data  
- Users without permission can see the button but cannot use it  
- Show warning message when unauthorized users attempt export  
- Ensure exported data respects user access permissions

### [TP] [DELIANY-3244](https://deliany.youtrack.cloud/issue/DELIANY-3244/BackOffice-Allow-Export-Person-Contacts-as-excel-file) [BackOffice] Allow Export Person Contacts as Excel file

**Issue**  
BackOffice users cannot export CRM Persons data to Excel, making it difficult to review, share, and analyze customer information outside the system.

**Solution**  
- Add **Export Excel** button on CRM Persons page  
- Export data based on current filters/search conditions  
- Generate `.xlsx` file with naming format: `crm_persons_data_YYYYMMDD_HHMMSS.xlsx`  
- Include required fields: name, phone, email, address, source, owner, etc.  
- Ensure:
  - Empty values remain blank  
  - Data matches system at export time  
  - Handle large data with loading indicator  
  - Prevent duplicate export actions while processing  
  - Show message if no data available  
  - Only export data user is authorized to access  

  ### [TP] [DELIANY-2320](https://deliany.youtrack.cloud/agiles/131-3/current?issue=DELIANY-2320) [BackOffice] Freemium onboarding set-up: Welcome screen

**Issue**  
Freemium users do not have a clear onboarding entry point after sign-up, leading to confusion about account creation status and next steps.

**Solution**  
- Redirect user to Welcome Onboarding screen after first login  
- Display onboarding steps (Welcome, Brand, Location) with current step highlighted  
- Show welcome content: illustration, success message, and CTA **“Let’s Get Started”**  
- Navigate to Brand setup after clicking CTA  
- Display user email, allow Logout and Language switch (EN/VI)  
- Preserve onboarding progress on refresh or re-login  
- Restrict access:
  - Only signed-up freemium users can access  
  - Completed users are redirected to BackOffice home  
- Handle invalid session by redirecting to safe page (Login/Landing)

### [TP] [DELIANY-2321](https://deliany.youtrack.cloud/agiles/131-3/current?issue=DELIANY-2321) [BackOffice] Freemium onboarding set-up: Brand

**Issue**  
There is no step for users to define brand identity during onboarding, which is required for system personalization and future operations.

**Solution**  
- Provide Brand Setup screen in onboarding flow  
- Include fields:
  - Brand Name (required)  
  - Description (optional)  
  - Logo upload (optional)  
- Validate Brand Name before allowing next step  
- Support logo upload (JPG, PNG, etc.) with preview and remove option  
- Keep user input when validation fails  
- Save data (name, description, logo) after successful submission  
- Restore saved data when user returns to this step  
- Only allow progress when required fields are valid

### [TP] [DELIANY-2311](https://deliany.youtrack.cloud/issue/DELIANY-2311/Freemium-Sign-up-Sign-up-form) [BackOffice] Freemium Sign-up: Sign-up form

**Issue**  
Users need a simple entry point to register for the freemium version, but current flow lacks a structured and validated sign-up process.

**Solution**  
- Provide sign-up form accessible via freemium link  
- Step 1: Input email  
  - Validate email format  
  - Check existing account → redirect to verify email if exists  
- Step 2: Input business information:
  - Fullname  
  - Password  
  - Business name  
  - Phone number  
  - Address  
- Email field is read-only in step 2  
- Submit form → send verification email  
- Ensure validation rules are applied before submission  
- Guide user through clear step-by-step registration flow
