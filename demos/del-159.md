## Sprint DEL-159

### [QH] Enhance and upgrade combo processing for Back Office, POS, and KDS.

Back-Office:
- Fixed an issue where the selected product list was cleared when reopening and saving the modal without changes.
- Added SKU details under product names in the `Select Product` modal to prevent confusion between identical item names.
- Enabled search functionality for combo names and combo SKUs.
- Resolved an issue preventing order updates on the Edit Order page when a combo contains additions.

POS:
- Applied the default prep station behavior to combo items when they are not pre-configured.
- Sorted combo items by their setup priority across Order Details, View Combo modal, Bill Printing, and KDS.
- Fixed an issue where combos couldn't be deleted while in the `New` state.
- Fixed the active/inactive configuration for combos so that only active combos are displayed on the POS listing.
- Fixed an issue where the `Save & Send` button was incorrectly disabled on the Edit Order page.

KDS:
- Fixed an issue where item checkboxes within a product were unclickable.
- Fixed a calculation bug where product quantities in printing were incorrectly inverted.
- Fixed an issue where a blank ticket was incorrectly sent to the Bar station when ordering combos routed entirely to the Kitchen.

### [QH] [DELIANY-3259](https://deliany.youtrack.cloud/issue/DELIANY-3259) Manage Reservations in Calendar Chart View

- Implemented the Calendar Chart (Gantt View) for the `Manage Reservations` feature.
- Added mini-calendar navigation with real-time summary cards for total Bookings and Guests.
- Displayed upcoming or active walk-in/reservation lists under the mini-calendar.
- Structured the reservation timeline grid grouped by table areas (e.g., Unassigned, Khu b) and table names.
- Implemented dynamic hourly timeline tracking (from 09:00) across the grid.
- Enabled dynamic reservation blocks (e.g., WALK IN) mapped onto specific tables and time slots with status colors.
